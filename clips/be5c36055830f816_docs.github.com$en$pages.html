        GitHub Pages 文档

        了解如何直接从 GitHub 上的存储库创建网站。探索 Jekyll 等网站构建工具，并解决 GitHub Pages 站点的问题。

        快速入门 概述

        从这里开始

        • 为 GitHub Pages 站点配置发布源

          可以将 GitHub Pages 站点配置为在将更改推送到特定分支时发布，也可以编写 GitHub Actions 工作流来发布站点。

        • 创建 GitHub Pages 站点

          可以在新的或现有的存储库中创建 GitHub Pages 站点。

        • 管理 GitHub Pages 站点的自定义域

          可以设置或更新某些 DNS 记录和存储库设置，以将 GitHub Pages 站点的默认域指向自定义域。

        流行

        • 关于自定义域和 GitHub Pages

          GitHub Pages 支持使用自定义域，或将站点 URL 的根目录从默认（如 octocat.github.io）更改为您拥有的任何域。

        • 关于 GitHub Pages 和 Jekyll

          Jekyll 是一个静态站点生成器，内置了对 GitHub Pages 的支持。

        • 对自定义域和 GitHub 页面进行故障排除

          可以检查常见错误，以解决 GitHub Pages 站点的自定义域或 HTTPS 问题。

        • 使用 HTTPS 保护 GitHub Pages 站点

          HTTPS 添加了一层加密，可防止其他人窥探或篡改您网站的流量。您可以为 GitHub Pages 站点强制实施 HTTPS，以透明地将所有 HTTP 请求重定向到 HTTPS。

        指南

        • 为 GitHub Pages 站点创建自定义 404 页面

          当用户尝试访问您网站上不存在的网页时，您可以显示自定义 404 错误页面。

          @GitHub

        • 使用 Jekyll 将主题添加到 GitHub Pages 站点

          您可以通过添加和自定义主题来个性化您的 Jekyll 网站。

          @GitHub

        • 排查 GitHub Pages 站点的 Jekyll 生成错误

          可以使用 Jekyll 生成错误消息来解决 GitHub Pages 站点的问题。

          @GitHub
